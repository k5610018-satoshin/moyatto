# モヤット 配布用セットアップ

このアプリのソースには Gemini API キーを入れません。学校や先生ごとに、最初に自分のキーを設定して使います。

## 1. GAS をコピーしてデプロイ

1. Google Apps Script にこのプロジェクトをコピーします。
2. 「デプロイ」からウェブアプリとして公開します。
3. 実行ユーザーは「自分」、アクセスできるユーザーは授業で使う範囲に合わせます。児童がログイン不要で使う場合は「全員」にします。

## 2. 先生用の合言葉を設定

デプロイ後のURLに `?admin=setup` を付けて開きます。

例:

```text
https://script.google.com/macros/s/デプロイID/exec?admin=setup
```

初回だけ、先生用の合言葉を決める画面が出ます。8文字以上で、他人が推測しにくい言葉にしてください。

設定後に表示される先生用URLを使うと、セッション作成、APIキー設定、結果表示のON/OFFができます。

## 3. Gemini API キーを設定

1. 先生用URLを開きます。
2. 「設定・セッション」を開きます。
3. Google AI Studio で作った Gemini API キーを貼り付けます。
4. 「保存」してから「テスト」を押し、AIが応答することを確認します。

APIキーは GAS のスクリプトプロパティにだけ保存され、児童の画面には表示されません。

## 4. 授業で使う

1. 先生画面でセッションを作ります。
2. 児童用URLを配ります。
3. 必要に応じて、先生画面でモードを固定します。
4. 「いいなと思われた見方」を児童に見せるときは、先生画面の「児童に結果を公開」をONにします。
5. 名前を隠したいときは「名前を表示」をOFFにします。

## 配布時の注意

- 自分の Gemini API キーをソースコードや配布ファイルに書かないでください。
- 先生用URLの合言葉は、児童に共有しないでください。
- リフレーミングは「我慢させる練習」ではありません。困ったままなら先生や友だちに相談してよいこと、次にできる行動も考えることを授業内で必ず扱ってください。
